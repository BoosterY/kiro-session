✂️ Topic Split

One long session, multiple topics? Split by meaning, resume just the part you need.

Semantic Grouping

LLM groups turns by meaning, not sequence. If you discussed Docker at turns 0-5 and returned at turns 11-15, they stay together.

🧠 LLM analyzes all turns
→
📋 Topic groups in index
→
📄 Temp file on resume
→
🔗 Derivation auto-tracked

Non-destructive

  • Topic mappings stored in the index only — kiro DB is never modified
  • Temp files generated on-demand when you choose to resume a topic
  • You load the topic in kiro-cli via /chat load — it becomes a new independent session

Derivation Tracking

When you load a topic file in kiro-cli, the new session is automatically linked back to its source.

🔗

Automatic

Source markers in temp files are detected on next index scan. No manual linking needed.

🌳

Multi-Hop Chains

Split a derived session again — the chain traces back to the original root session.

📦

Fully Derived

When all topics have derived sessions, the source is marked for archival in cleanup.

🔓

Independent

Derived sessions are fully independent. Deleting one never affects the other.

Large Sessions

Chunk-Analyze-Merge (50+ turns)

  • Splits turns into ~50-turn chunks, each analyzed independently
  • Merge pass: LLM reviews all chunk topics, merges duplicates, deduplicates turn indices
  • A 500-turn session processes in chunks — no context window limits
